# Claude Code 命令指南

> 适合：刚接触 Claude Code 的小白。从日常使用到进阶技巧，一页搞定。

---

## 一、先认识你的界面

```
┌─────────────────────────────────────────────────┐
│  Claude Code · model: sonnet · mode: default    │  ← 状态栏
├─────────────────────────────────────────────────┤
│                                                 │
│  对话区域                                        │
│  (你和 Claude 的聊天记录)                         │
│                                                 │
├─────────────────────────────────────────────────┤
│  > 在这里输入                                     │  ← 输入区
└─────────────────────────────────────────────────┘
```

三个最重要的状态信息：
- **model**：当前使用的模型（sonnet/opus/haiku）
- **mode**：权限模式（default/acceptEdits/plan/bypassPermissions）
- **分支名**：当前 Git 分支

## 二、基础操作（第一天就要会的）

### 2.1 启动 Claude Code

```bash
# 在当前目录启动（最常用）
claude

# 直接问一个问题然后退出（不进入交互模式）
claude -p "这个项目是做什么的"

# 继续上次的会话
claude --continue

# 指定模型启动
claude --model opus
```

### 2.2 三个最常用前缀

在输入框里直接用：

| 输入 | 作用 | 举例 |
|------|------|------|
| `!` | 直接执行终端命令 | `! ls` 查看文件列表 |
| `/` | 执行斜杠命令 | `/help` 查看所有命令 |
| `@` | 引用文件（有自动补全） | `@src/main.py` 让 Claude 看这个文件 |

### 2.3 和 Claude 交流的基本技巧

```bash
# 直接问
> 这个文件夹里有什么文件？

# 引用文件
> @src/main.py 这个文件做了什么？

# 让它做事
> 帮我在 README.md 里添加安装说明
```

## 三、斜杠命令速查（`/` 开头）

### 🔥 日常必备

| 命令 | 作用 | 什么时候用 |
|------|------|-----------|
| `/help` | 查看所有命令和快捷键 | 忘了命令的时候 |
| `/clear` | 清空对话，重新开始 | 话题换了，别让旧对话干扰 |
| `/compact` | 压缩对话，释放上下文 | 聊太久了，Claude 开始"忘事" |
| `/cost` | 查看花了多少钱 | 怕超预算 |
| `/context` | 查看上下文用了多少 | Claude 说"内存不够"时 |
| `/rename 名字` | 给当前会话起个名 | 方便之后找到它 |
| `/resume` | 打开历史会话列表 | 想回到之前的对话 |

### 🛠 代码相关

| 命令 | 作用 |
|------|------|
| `/init` | 分析项目，生成 CLAUDE.md 项目说明文件 |
| `/diff` | 查看你改了什么代码（带 AI 总结） |
| `/commit` | 自动生成 commit 信息并提交 |
| `/code-review` | 代码审查，帮你找 Bug |
| `/simplify` | 优化当前改动的代码 |
| `/review` | 审查 Pull Request |

### ⚙️ 设置相关

| 命令 | 作用 |
|------|------|
| `/config` | 打开可视化设置面板 |
| `/model` | 切换模型（sonnet/opus/haiku） |
| `/theme` | 切换颜色主题 |
| `/permissions` | 管理权限设置 |
| `/vim` | 开启/关闭 Vim 编辑模式 |

### 📋 工作模式

| 命令 | 作用 |
|------|------|
| `/plan` | 进入规划模式（只能看，不能改） |
| `/todos` | 查看当前任务列表 |
| `/agents` | 管理后台子任务 |
| `/loop 5m 命令` | 每 5 分钟自动执行一次命令 |

## 四、权限模式（`Shift+Tab` 切换）

按下 `Shift+Tab` 可以在四种模式间循环切换，状态栏会显示当前模式：

| 模式 | 状态栏 | 行为 | 推荐场景 |
|------|--------|------|----------|
| **default** | (无图标) | 所有操作都要你确认 | 刚开始用，不了解的时候 |
| **acceptEdits** | `>>` | 改文件自动通过，执行命令要确认 | 🔥 **日常开发推荐** |
| **plan** | 📊 | 只能看和搜索，不能改 | 想看方案，不想动手 |
| **bypass** | `>>` | 全部自动执行 | 完全信任 Claude 时 |

> 💡 **新手建议**：前几次用 default 模式，熟悉了切到 acceptEdits，效率翻倍。

## 五、键盘快捷键

### 最常用的几个

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+C` | 停止 Claude 当前输出 |
| `Ctrl+D` | 退出 Claude Code |
| `Ctrl+L` | 清屏（对话还在） |
| `Ctrl+R` | 搜索输入历史 |
| `Shift+Tab` | 切换权限模式 |
| `Ctrl+B` | 查看后台运行的任务 |
| `Ctrl+G` | 在外部编辑器打开当前输入 |

### 权限确认弹窗

| 按键 | 含义 |
|------|------|
| `y` 或 `Enter` | ✅ 同意这个操作 |
| `n` | ❌ 拒绝这个操作 |
| `a` | ✅ 同意本次会话的所有同类操作 |
| `Esc` | 取消 |

### 文本编辑（Emacs 风格）

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+A` | 跳到行首 |
| `Ctrl+E` | 跳到行尾 |
| `Ctrl+K` | 删到行尾 |
| `Ctrl+W` | 删前一个单词 |
| `Ctrl+Y` | 粘贴刚才删的 |
| `↑↓` | 浏览输入历史 |

### 多行输入

| 方法 | 说明 |
|------|------|
| `\` + `Enter` | **最通用的方法**，换行 |
| `Shift+Enter` | 大多数终端可用 |
| `Ctrl+J` | 所有终端都能用 |

## 六、常用 CLI 启动参数

```bash
# === 会话管理 ===
claude                                    # 新建会话
claude --continue                         # 继续最近会话
claude --resume                           # 打开历史会话列表
claude --resume "会话名"                   # 恢复指定会话
claude --name "我的任务"                   # 启动时给会话起名

# === 模型选择 ===
claude --model sonnet                     # 用 Sonnet（默认，日常够用）
claude --model opus                       # 用 Opus（最强，复杂任务）
claude --model haiku                      # 用 Haiku（最快最便宜，简单任务）

# === 一行模式 ===
claude -p "这段代码做什么的"               # 问完就结束，不进入交互
claude -p "解释项目结构" --output-format json  # 输出 JSON 格式

# === 权限 ===
claude --permission-mode acceptEdits      # 启动就用 acceptEdits 模式
claude --permission-mode plan             # 启动就用 plan 模式
```

## 七、新手常见场景

### 场景 1：让 Claude 分析一个项目

```bash
cd 你的项目目录
claude
> /init          # 让 Claude 分析项目并生成说明文件
> 帮我总结这个项目的架构
```

### 场景 2：让 Claude 帮你写代码

```bash
> 在 src/utils.py 里添加一个函数，功能是读取 CSV 文件并返回行数
> @src/utils.py 给这个函数加上类型注解
```

### 场景 3：让 Claude 帮你修 Bug

```bash
> @src/main.py 运行报错 "KeyError: 'name'"，帮我看看怎么修
```

### 场景 4：代码审查

```bash
> /diff          # 先看看改了什么
> /code-review   # 让 Claude 审查你的改动
```

### 场景 5：聊太久了，上下文不够

```bash
> /context       # 先看看用了多少
> /compact       # 压缩一下，腾出空间
# 或者：
> /clear         # 直接清空重新开始
```

### 场景 6：找到之前的对话

```bash
> /resume        # 打开历史会话列表
# 用 ↑↓ 选择，Enter 进入
# 输入 / 可以搜索
```

## 八、进阶技巧

### 8.1 CLAUDE.md — 你的项目说明书

在项目根目录创建 `CLAUDE.md`，Claude 每次启动都会读它：

```markdown
# CLAUDE.md

## 项目简介
这是一个银行风控系统，用 Python + FastAPI 构建。

## 编码规范
- Python 3.10+
- 用 pytest 写测试
- 提交信息用中文

## 注意事项
- 不要改 config/ 目录下的配置文件
- 数据库是 PostgreSQL
```

用 `/init` 可以自动生成。

### 8.2 自定义斜杠命令

在 `.claude/commands/` 下创建 Markdown 文件：

```markdown
---
description: "运行所有测试"
---

运行 pytest，显示详细输出。如果有失败的测试，分析原因。
```

然后就可以用 `/运行所有测试` 来调用了。

### 8.3 管道 + 一行模式

```bash
# 让 Claude 分析日志
cat error.log | claude -p "分析这些错误日志，找出根本原因"

# 让 Claude 解释代码
cat main.py | claude -p "解释这段代码的功能"
```

### 8.4 后台子任务

```bash
> /fork 帮我在后台检查这个项目的安全问题
# 你继续做你的事，后台任务完成后会通知你
```
